# 벨트 장력계 2.0

벨트 장력계 v1을 기반으로 다중 로봇 관리·PDF 보고서·벨트 ID 체계를 추가한 2.0 버전입니다.
CSS·JS를 별도 파일로 분리한 구조입니다.

## v1 대비 추가된 기능

- **다중 로봇·유닛 관리**: 로봇을 등록하고 유닛별 벨트를 순서대로 측정합니다.
- **5단계 측정 플로우**: 로봇 등록 → 벨트 선택 → 측정 → 저장 → 완료 단계로 진행합니다.
- **벨트 ID 관리**: ID1~ID6 (구동모듈 4개·리프팅 2개) 고정 ID를 선택해 측정합니다.
- **PDF 보고서**: jsPDF 기반으로 측정 결과를 PDF로 출력합니다.
- **단순 측정 모드**: 플로우 없이 즉시 측정하는 모드도 지원합니다.
- **파일 분리**: `index.html` + `style.css` + `app.js` 3파일 구조로 유지보수가 편합니다.

## 원리

1. 마이크가 벨트를 두드리는 소리를 녹음합니다.
2. Web Audio API AnalyserNode가 실시간 FFT를 수행합니다.
3. 지배 주파수 `f`와 벨트 파라미터(단위 질량, 폭, 스팬)로 장력을 계산합니다:

   ```
   T [N] = 4 × m × w × L² × f² × 1e⁻⁹
   ```

## 주요 기능

- 실시간 FFT 주파수 검출 (10 ~ 5000 Hz)
- 장력 자동 계산 (보정계수 CF 적용)
- 목표 장력 게이지 (허용 오차 시각화, OK/NG 판정)
- 노이즈 저감 필터 (스펙트럴 서브트랙션)
- 목표 주파수 잠금 모드
- 프리셋: 리프팅 벨트(500 N ±10%) / 구동모듈 벨트(170 N ±20%)
- 측정 기록 누적 및 CSV·PDF 내보내기

## 파라미터

| 상수 | 값 | 설명 |
|---|---|---|
| `MIN_FREQ` | 10 Hz | 탐색 하한 주파수 |
| `MAX_FREQ` | 5000 Hz | 탐색 상한 주파수 |
| `BUF_SIZE` | 5 | 이동 평균 버퍼 크기 |
| `OUTLIER_TOL` | 0.12 | 이상치 허용 비율 |
| `G` | 9.80665 | 중력 가속도 (m/s²) |

## 벨트 ID

| ID | 종류 | 목표 장력 | 허용 오차 |
|---|---|---|---|
| ID1 ~ ID4 | 구동모듈 벨트 | 170 N | ±20% |
| ID5 ~ ID6 | 리프팅 벨트 | 500 N | ±10% |

## 데이터 저장

측정 기록은 `localStorage['beltTension_v1']`에 JSON으로 저장됩니다.
메인 포털 → 데이터 관리에서 초기화할 수 있습니다.
